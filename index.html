Welcome to my 'URL shortener MS' page

This is a microservice which shortens the new URLs that are passed in, and redirects requests sent to these shortneed URLs to the right URLs

Example usage:

  1. Pass in a new URL to be stored, and get back the shortened URL corresponding to this URL: (pass with a '/new/' route specified in the URL) 
            e.g:
        https://warm-reef-87592.herokuapp.com/new/http://www.freecodecamp.com/
        {"original_url":"http://www.freecodecamp.com/","short_url":"https://warm-reef-87592.herokuapp.com/1482780504987"}
  2. Use one of the existing shortened URLs to get redirected to the corresponding page 
            e.g:
        https://warm-reef-87592.herokuapp.com/1482780504987
        will redirect to 
        http://www.freecodecamp.com
  3. Use one of the non-existing short URLs (i.e. specify anumber which doesn't exist in the db) This will elicit an error response from this microservice 
            e.g:
        https://warm-reef-87592.herokuapp.com/14827805049875345353453
        {"error":"This url is not present in the database."}
  4. Any other URL patterns than the ones specified above, are not being listened for, and will result in errors.

Following user stories are covered in this microservice:

  • User Story 1: I can pass a string as a parameter, and it will check to see whether that string contains either a unix timestamp or a natural language date (example: January 1, 2016).
  • User Story 2: If it does, it returns both the Unix timestamp and the natural language form of that date.
  • User Story 3: If it does not contain a date or Unix timestamp, it returns null for those properties.
